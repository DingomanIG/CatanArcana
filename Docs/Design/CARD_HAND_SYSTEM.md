# 카드 핸드 시스템 설계

## 개요
Balatro 스타일 부채꼴 싱글 핸드로 자원카드, 발전카드, 보너스카드를 통합 관리한다.
카테고리별 그룹핑 + 색상/테두리로 시각적 구분.

---

## 핸드 구조

### 싱글 핸드 — 통합 배치
- **위치**: 화면 하단 중앙
- **배치 순서** (좌 → 우):
  ```
  [자원 카드] | [발전 카드] | [보너스 카드]
   목재 벽돌 양모 밀 광석 | 기사 도로건설 풍년 독점 승리점 | ★최장도로 ★최강기사
  ```
- **카테고리 구분**: 그룹 간 간격(spacing) + 카드 테두리 색상으로 시각적 분리
  - 자원카드: 녹색 계열 테두리
  - 발전카드: 보라색 계열 테두리
  - 보너스카드: 금색 계열 테두리 + 별 아이콘
- **정렬 규칙**: `CardData.SortOrder` 기반, 카테고리 내 고정 순서
  - 드래그 스왑 비활성화 (정렬 순서 유지)

### 자원 카드 스택킹 (팬 스택 방식)
- 같은 자원 다수 보유 시 **카드들이 2~3px씩 어긋나며 겹침** + 숫자 배지
  - 예: 목재 x3 → 3장이 살짝 어긋나게 겹침, 맨 앞 카드에 "x3" 배지
  - 뒤에 깔린 카드 모서리가 보여 "여러 장" 느낌 유지
- 스택 오프셋: 카드당 3px 수평 + 1px 수직
- 핸드 과밀 방지 (자원 10장이 5스택으로 압축)
- 호버 시 스택 전체가 살짝 펼쳐지며 수량 + 자원 설명 표시

### 카테고리별 인터랙션

| | 자원카드 | 발전카드 | 보너스카드 |
|--|---------|---------|-----------|
| 호버 | 확대 + 자원 설명 | 확대 + 효과 설명 | 확대 + 설명 |
| 클릭 | 선택 (거래/건설) | 선택 | - |
| 드래그 위로 | - | 사용 시도 | 불가 |
| 다중 선택 | 도적 디스카드 시 | - | - |

## 카드 생성/제거 연출

### 자원 카드
- **획득**: 타일 월드 좌표 → 스크린 좌표 변환 → 핸드 목표 위치로 날아옴
  - 스택에 추가되는 경우: 기존 스택으로 날아가며 뒤에 겹침 + 배지 증가
- **소모**: 핸드에서 위로 날아가며 사라짐 (거래/건설 시)
  - 스택에서 감소: 뒤 카드 제거 + 배지 감소 연출, 0이면 스택 전체 제거
- **디스카드**: 선택된 카드들이 화면 중앙(도적)으로 날아감

### 발전 카드
- **구매**: 덱 UI 위치 → 핸드 목표 위치로 날아옴
- **사용**: 드래그 후 상단 임계선 초과 시 → 화면 중앙으로 확대 → 스케일 바운스 → 효과 발동
- **사용 불가**: 좌우 흔들림(shake) → 원래 위치로 OutBack 복귀

### 보너스 카드 (최장도로/최강기사)
- **획득**: 화면 상단에서 핸드 오른쪽 끝으로 드로우
- **상실**: 핸드에서 위로 날아가며 사라짐 (다른 플레이어가 탈취 시)

---

## 카드 사용 플로우 (발전카드)

```
드래그 시작
  → 카드가 마우스를 따라감
  → 상단 임계선(화면 60% 높이) 초과?
    ├─ YES: 사용 가능 턴인가?
    │   ├─ YES → 카드 사용 확정, 위로 날아가며 효과 발동
    │   └─ NO  → 좌우 shake 연출, 핸드로 복귀
    └─ NO: 마우스 놓으면 핸드로 복귀
```

---

## 도적 디스카드 모드

```
7 초과 자원 보유 시 활성화
  → 핸드 "다중 선택 모드" 진입
  → 자원 카드만 클릭 가능 (발전/보너스 비활성)
  → 카드 클릭 시 토글 선택 (선택 시 위로 올라감)
    - 스택 카드: 클릭마다 선택 수량 +1, 배지로 선택 수/보유 수 표시
  → 필요 디스카드 수 충족 → 확인 버튼 활성화
  → 확인 → 선택 카드들이 도적 위치로 날아감
  → 모드 해제
```

## 호버 확대 상세

| 항목 | 값 |
|------|-----|
| 확대 비율 | 1.3x ~ 1.5x |
| 위로 이동 | 카드 높이의 50% |
| 전환 시간 | 0.15s (OutBack) |
| 설명 표시 | 카드 상단에 툴팁 or 카드 자체 확대로 텍스트 가독 |
| 주변 카드 | 좌우로 살짝 밀려남 (공간 확보) |

## 구현 시 주의사항

### 다중 선택 모드 (도적 디스카드)
- `CardHandManager`에 `SelectionMode` 상태 추가
  - `Normal`: 단일 선택 (기본)
  - `MultiSelect_Discard`: 도적 디스카드용 복수 선택
- 모드 진입 시 발전/보너스 카드 인터랙션 비활성화
- 모드 해제 시 선택 상태 전체 초기화

### 카드 사용 임계선
- "화면 60% 높이"는 절대 px가 아닌 **RectTransform 비율 기반**으로 계산
- Canvas 해상도 독립적으로 동작하도록 구현

### 자원카드 동기화
- `PlayerState` 자원 변화 → 스택 카드 수량 자동 연동
- `PlayerState`에 자원 변경 이벤트 콜백 추가 (옵저버 패턴)

### 팬 스택 고려사항
- 스택 = 대표 카드(맨 앞) + 뒤에 어긋난 카드들 (시각용, 개별 인터랙션 없음)
- 호버 시: 스택 전체가 살짝 펼쳐지며 수량 정보 표시
- 선택 시 (디스카드 모드): 클릭마다 선택 수량 증감, 최대 보유 수 제한
- 스택 오프셋: 카드당 3px 수평, 1px 수직 (최대 표시 5장까지)

## 기술 참고
- 기반: `Assets/Scripts/UI/CardHand/` (Balatro 프로토타입)
- UI: UGUI (Canvas) + DOTween
- 좌표 변환: `Camera.WorldToScreenPoint` → `RectTransformUtility.ScreenPointToLocalPointInRectangle`
- 데이터 소스: `PlayerState` (자원 배열), `DevelopmentCard` (발전카드 리스트)
- 정렬: `CardData.SortOrder` — 카테고리 내 고정 순서, 카테고리 간 그룹핑
