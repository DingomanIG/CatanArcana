# 멀티플레이 동기화 테스트 목록

> 테스트 도구: ParrelSync (원본=호스트, 클론=클라이언트)  
> 테스트 우선순위: A → B → C → D → E → H → F → G → I → J → K → L → M → N → O → P

## A. 게임 시작 & 보드 동기화

- [x] **A1** 로비 → 게임 씬 전환 — 양쪽 모두 Game 씬 진입
- [x] **A2** 보드 생성 & 스냅샷 전송 (`BoardSnapshot`) — 클라이언트 보드 = 호스트 보드
- [x] **A3** 플레이어 인덱스 할당 (`AssignPlayerIndexClientRpc`) — 각 클라이언트가 자기 인덱스 인식
- [x] **A4** 플레이어 이름 표시 (`netPlayerNames`) — 양쪽에서 상대방 이름 보임
- [x] **A5** 턴 순서 오버레이 (`ShowTurnOrderClientRpc`) — 양쪽 동일한 순서 표시

## B. 초기 배치 (Initial Placement)

- [x] **B1** 1라운드 정방향 (1→2→3→4) — 올바른 순서로 배치 진행
- [x] **B2** 2라운드 역방향 (4→3→2→1) — 스네이크 드래프트 정확히 작동
- [x] **B3** 정착지 배치 → 상대방 화면 반영 (`NotifyBuildingPlacedClientRpc`)
- [x] **B4** 도로 배치 → 상대방 화면 반영 (`NotifyRoadPlacedClientRpc`)
- [x] **B5** 2라운드 정착지 초기 자원 지급 (`NotifyResourceUpdateClientRpc`)
- [x] **B6** 거리 규칙 위반 시 거부 — 인접 꼭짓점에 배치 불가

## C. 주사위 & 자원 분배

- [ ] **C3** 사막/강도 타일 미생산 — 강도 있는 타일 자원 안 줌

## D. 강도 (7 굴렸을 때)

- [X] **D1** 8 이상 자원 보유자 반토막 (`NotifyDiscardRequiredClientRpc`) — 8장 이상만 디스카드 요청
- [X] **D2** 디스카드 확인 (`RequestConfirmDiscardServerRpc`) — 선택 자원 차감, 게임 진행
- [X] **D3** 강도 이동 (`NotifyRobberMovedClientRpc`) — 강도 이동전에 상대가 디스카드 요청중이면 주사위를 굴림 그리고 도적 이동 ~~(수정: 디스카드 대기 중 RollDice 차단 추가)~~
- [ ] **D4** 도둑질 대상 선택 (`NotifyStealCandidatesClientRpc`) — 해당 타일 건물주만 후보
- [X] **D5** 자원 훔치기 실행 (`NotifyRobberStealClientRpc`) — 도둑/피해자 자원 변동
- [X] **D6** 강도 위치에서 자원 미생산 — 이후 주사위에서 해당 타일 자원 차단
